# 백업 전략

홈랩 인프라를 두 계층으로 나눠 백업합니다. 하나는 VM 전체, 하나는 앱 데이터(PVC)입니다.

## 두 계층 백업

**1. 인프라 계층 — PBS (Proxmox Backup Server)**
- 대상: Proxmox에서 도는 VM·LXC 전체
- 저장소: TrueNAS NFS
- 주기: 야간 1회
- 용도: 노드가 통째로 망가졌을 때(OS 부팅 불가 등) VM 자체를 특정 시점으로 복원

**2. 애플리케이션 계층 — Longhorn 백업**
- 대상: 파드의 영속 데이터(PVC)
- 저장소: SeaweedFS(S3 호환 오브젝트 스토리지)
- 용도: 특정 앱 데이터만 되돌리거나, 다른 환경으로 앱을 옮길 때

추가로 K3s etcd 스냅샷도 SeaweedFS에 저장해 클러스터 상태를 보관합니다.

## 저장소 구성

SeaweedFS는 TrueNAS의 ZFS 풀을 마운트해 S3 호환 스토리지로 씁니다. 백업 용도별로 **버킷을 나누고, 각 버킷에 전용 계정을 만들어 해당 버킷에만 권한을 부여**합니다(최소 권한 원칙).

| 백업 | 저장소 | 스케줄 |
|------|--------|--------|
| PBS VM/LXC | TrueNAS NFS | Proxmox WebUI, 야간 1회 |
| Longhorn 볼륨 | SeaweedFS(S3) | Longhorn WebUI |
| etcd 스냅샷 | SeaweedFS(S3) | K3s 내장 |

> 두 백업이 동시에 돌면 디스크·네트워크가 병목될 수 있어, 시간대를 겹치지 않게 분리하는 게 좋습니다.

## 복구 방향 (설계)

> 아래는 백업을 전제로 정리한 복구 방향입니다. 실제 재해 복구를 수행한 이력은 아직 없고, 백업이 정상 동작·저장되는 것까지 확인했습니다. 복구 리허설은 향후 과제입니다.

- **특정 앱 데이터 손상**: PBS로 VM 전체를 되돌리면 다른 앱까지 과거로 가버리므로, Longhorn에서 해당 PVC만 이전 스냅샷으로 복원.
- **워커 노드 OS 손상**: PBS에서 해당 VM을 복원. K3s가 다시 클러스터에 합류.
- **역할 분리**: 앱 데이터는 Longhorn으로, 시스템 레벨 장애는 PBS로 대응하는 식으로 나눠서 운영합니다.
